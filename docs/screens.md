## 2. 各画面の詳細

### 画面 1：ログイン（`/login`）

**ファイル：** `Login.jsx`

**表示内容：**
- 左パネル（PC のみ）：背景画像、ロゴ、「クレームは、成長の種。」キャッチコピー、機能カード4枚
- 右パネル：メールアドレス / パスワード入力、ログインボタン

**実装済み操作：**
- メール＋パスワードでログイン（`supabase.auth.signInWithPassword`）
- パスワード表示/非表示トグル
- Enter キーでログイン実行

**未実装（UI のみ）：**
- 「ログインしたままにする」チェックボックス → `rememberMe` state はあるが Supabase に渡されていない
- 「パスワードをお忘れの方」ボタン → 処理なし
- 「新規登録はこちら」ボタン → 処理なし

---

### 画面 2：ダッシュボード（`/dashboard`）

**ファイル：** `Dashboard.jsx`

**表示内容（上から順）：**

1. **ヘッダー行**：「クレーム管理ダッシュボード」タイトル、「＋ 新規依頼」ボタン

2. **統計カード（5枚）：**
   - 未対応クレーム数（承認完了除く）
   - 本日の受付件数
   - 期限超過件数（赤表示）
   - 今月のクレーム件数（先月比）
   - 先月のクレーム件数

3. **タブ（3種）：**
   - 全件
   - 自分の担当（※ 現在は `isMine = false` で全件と同一）
   - 全スタッフのクレーム一覧

4. **表示モード切替：**
   - 通常表示：タイマー付き ComplaintCard（感情レベル色・ステータスフィルター対応）
   - ステータス表示：StatusComplaintCard（ステータス順ソート）

5. **ステータスフィルター：**（通常表示時のみ）
   未対応 / 受付済 / 対応中 / 是正案提出 / 是正案差し戻し / 是正案承認 / 改善報告書提出 / 深掘り提出 / 承認完了

6. **クレーム一覧**

7. **掲示板セクション（「🌱 改善報告書掲示板」）：**
   - キーワード検索（クレーム内容・現場名・真因・組織対策を対象）
   - 真因カテゴリーフィルター（8分類 + 全て、複数選択可）
   - 期間フィルター（全て / 直近1ヶ月 / 直近3ヶ月 / 直近6ヶ月 / 1年以内）
   - 件数表示（「3件 / 10件中」形式）
   - BulletinCard 一覧（新着順）

**タイマー：** 1秒ごとに再レンダリング（`setInterval`）

### 画面 3：新規クレーム受付（`/complaints/new`）

**ファイル：** `ComplaintNew.jsx`

**入力項目：**

| フィールド | 種別 | 備考 |
|-----------|------|------|
| 受付日時 | 自動入力 | フォーム表示時の日時を固定 |
| 元請様名 | テキスト | |
| 元請担当者様名 | テキスト | |
| 現場名 | テキスト | |
| 現場作業者名 | テキスト | |
| 作業に入った日 | 日付 | |
| カテゴリ | ボタン選択 | 破損 / 施工不備 / 遅刻 / マナー / 近隣トラブル / その他 |
| 詳細内容 | テキストエリア | 必須 |
| 感情レベル | ボタン選択（1〜5） | 対応期限に連動 |
| 担当部署 | セレクト | 選択で担当者を自動入力 |
| 担当者 | テキスト | 担当部署から自動入力・手動変更可 |
| 受付者名 | テキスト（読み取り専用） | ログインユーザーの `full_name` を自動入力 |

| 清掃部 清掃１課 | 井上参事 |
| 清掃部 清掃２課 | 備主任 |
| 工事部 解体課 | 松木主任 |
| 工事部 産廃課 | 新田主任 |
| 環境リサイクル部 | 岡橋次長 |
| 本部 | 榮藤取締役 |

**対応期限タイマー：** フォーム表示中リアルタイムカウントダウン（1秒更新）

### 画面 4：クレーム詳細概要（`/complaints/:id`）

**ファイル：** `ComplaintOverview.jsx`

**表示内容：**
- 7ステップ ProgressBar
- 基本情報カード（感情レベル・クレーム内容・担当者・ステータス）
- 以下の7セクション（カード形式）：

| # | セクション | 確認内容 |
|---|-----------|---------|
| ① | お客様への連絡記録 | contact ログ一覧・初回連絡タイマー |
| ② | 作業者からの聞き取り | hearing ログ一覧 |
| ③ | 是正案 | report ログの内容 |
| ④ | 事業責任者確認 | supervisor_comment ログ / complaint.supervisor_comment（重複防止あり） |
| ⑤ | 改善報告書（現象原因の特定） | correction の内容（直接原因・是正処置・運用改善案） |
| ⑥ | 深掘り分析 | deep_analysis の内容（真因・カテゴリー・横展開・真因対策・残日数） |
| ⑦ | 役員承認記録 | approvals 一覧（○/○名承認） |

**ロール別アクションボタン（ComplaintOverview 内に定義）：**

| ロール | 条件 | ボタン | 遷移先 |
|-------|------|-------|-------|
| admin | 受付済・対応中・差し戻し | 対応入力 → | `/detail` |
| admin | 是正案承認 | 改善報告書を作成 → | `/correction` |
| admin | 改善報告書提出 | 改善報告書を確認 → | `/correction` |
| manager | 是正案提出・差し戻し | 是正案を確認・承認 → | `/deep-analysis` |
| manager | 是正案承認・改善報告書提出 | 深掘り分析を入力 → | `/deep-analysis` |
| director | 是正案提出・差し戻し | 是正案を確認・承認 → | `/deep-analysis` |
| executive / judgment | 深掘り提出 | 役員承認へ → | `/approval` |
| executive / admin | 深掘り提出・未全員承認 | 合同改善報告書を確認する → | `/approval` |

### 画面 5：聞き取り・対応入力（`/complaints/:id/detail`）

**ファイル：** `ComplaintDetail.jsx`  
**アクセス制限：** `admin`, `judgment`

**機能：**
- 対応期限タイマー表示（初回連絡後は「初回連絡まで」に切替）
- ① お客様への連絡記録（「繋がらず」ボタン + 自由入力）
- ② 作業者からの聞き取り記録
- ③ 対応判断（「手直し」/ 「事業責任者へ確認」の2択）
  - 手直し：手直し予定日時を入力 → 上司に報告
  - 事業責任者：自分の考えをテキスト入力 → 上司に報告
- 是正案差し戻し時の修正UI
- 是正案承認後の是正報告書作成ボタン

---

### 画面 6：改善報告書（`/complaints/:id/correction`）

**ファイル：** `CorrectionSubmit.jsx`  
**アクセス制限：** `admin`, `judgment`

**機能：**
- 発生事実（連絡記録・聞き取り）の自動表示
- ② 直接原因の入力 → ソクラテス式対話を自動起動
- ③ 是正処置の入力
- ④ 運用改善案の入力（ソクラテス対話から自動反映）
- 提出ボタン → `/complaints/:id` へリダイレクト
- 再提出時は過去の回答・ソクラテス対話回答を復元

---

### 画面 7：深掘り分析（`/complaints/:id/deep-analysis`）

**ファイル：** `DeepAnalysisForm.jsx`  
**アクセス制限：** `manager`, `director`

**ステータスによって表示内容が変わる（同一 URL・同一ファイル）：**

| complaints.status | 表示内容 |
|-----------------|---------|
| `是正案提出` / `是正案差し戻し` | 是正案確認・承認/否認フォーム |
| `是正案承認`（correction なし） | 事業責任者コメント追加フォーム |
| `改善報告書提出` / `是正案承認`（correction あり） | 改善報告書確認 → 深掘り分析フォーム |
| `深掘り提出` | 合同改善報告書プレビュー + 役員提出確認 |

**提出期限タイマー：** `correction.created_at` から24時間のカウントダウン（赤・オレンジ・黄で段階表示）

---

### 画面 8：役員承認（`/complaints/:id/approval`）

**ファイル：** `Approval.jsx`  
**アクセス制限：** `judgment`, `executive`, `admin`

**表示内容：**
- 合同改善報告書プレビュー（クレーム概要・対応の顛末・事業責任者確認・改善報告書・深掘り分析）
- 事業責任者の深掘り結果（読み取り専用）
- 役員承認カード × 3名（承認/差し戻しボタン・コメント入力）
- 承認率プログレスバー
- 24時間承認期限カウントダウン
- 今月の根源テーマ集計グラフ（**未実装：全件対象・古い4分類**）
- 承認完了メッセージ（🌱 Seed Note が完成しました）

---

### 画面 9：マイページ（`/mypage`）

**ファイル：** `MyPage.jsx`

**機能：**
- ロール・メールアドレス表示
- 個人情報編集（担当者名・ふりがな・携帯電話番号）→ `profiles` テーブルへ upsert
- パスワード変更（`supabase.auth.updateUser`）
- ログアウト

## 5. ナビゲーション構造

```
ログイン画面（未認証）
    │
    ▼
ダッシュボード（/dashboard）
    │
    ├── ＋ 新規依頼 → /complaints/new → 登録後 /dashboard に戻る
    │
    └── クレームカード クリック
            │
            ▼
        クレーム詳細（/complaints/:id）
            │
            ├── [admin/judgment] 対応入力 → /complaints/:id/detail
            │                                  → /complaints/:id に戻る
            ├── [admin/judgment] 改善報告書 → /complaints/:id/correction
            │                                  → /complaints/:id に戻る
            ├── [manager/director] 深掘り → /complaints/:id/deep-analysis
            │                                  → /complaints/:id に戻る
            └── [executive/admin] 役員承認 → /complaints/:id/approval
                                              → /dashboard に戻る（完了時）

ヘッダー（全画面共通）
    └── ユーザー名 クリック → /mypage
```
